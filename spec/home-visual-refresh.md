# 全站视觉刷新需求文档

## 需求描述

当前站点前端页面大量使用蓝色、青色与蓝色渐变，首页首屏、头像光晕、打字条、按钮、卡片 hover、分区边界和其他页面的视觉语言都集中在同一套蓝色系上。整体识别度偏弱，蓝色渐变显得常见且略显廉价，不符合个人站希望呈现的专业、可信、有记忆点的形象。

本次需求先只调整首页，不调整全站页面。首页确认效果后，再把验证过的视觉语言扩展到关于、项目、服务、博客和联系等页面。

当前首页试点效果已确认，下一阶段将把首页已经验证通过的 `Editorial Tech / Warm Graphite` 视觉方案适配到整个网站。

注意：现有页面文案不做调整，包括标题、简介、按钮文字、项目说明、服务说明、时间线说明等。允许调整的是视觉设计、布局组织、组件样式、色彩系统、空间节奏、动效与交互反馈。

## 需求目标

- 去掉全站主要区域中过度使用的蓝色渐变，建立更成熟、更有辨识度的个人站视觉方向。
- 首页作为试点已经确认，后续全站适配需要沿用首页已验证的视觉语言，保证页面之间一致。
- 保持现有文案和信息结构的可读性，不因为设计调整降低内容传达效率。
- 使用 `frontend-design` 的要求执行：设计要有明确审美方向，避免通用 AI 风格，避免俗套蓝紫渐变，强调字体、色彩、空间、动效和细节一致性。
- 每个适配批次完成后必须运行 `npm run lint`、`npm test`、`npm run build`，确保前端质量。

## 设计方向

推荐方向：`Editorial Tech / Warm Graphite`

这是偏编辑化、偏专业顾问气质的个人站方向。整体避免蓝色科技模板感，用石墨黑、暖白、翡翠绿和琥珀金建立层次：

- 主背景：暖白到浅灰，不使用蓝色大面积渐变。
- 主文字：石墨黑，提高专业感和阅读稳定性。
- 主强调色：翡翠绿，用于姓名、关键数字、主按钮、焦点态。
- 辅助强调色：琥珀金，用于细节点缀、状态点、时间线或卡片强调。
- 深色模式：深石墨背景，不使用蓝色霓虹，改用低饱和绿与暖金。

建议色板：

| 用途 | 浅色模式 | 深色模式 |
| --- | --- | --- |
| Background | `oklch(0.975 0.006 95)` | `oklch(0.145 0.012 110)` |
| Foreground | `oklch(0.18 0.012 120)` | `oklch(0.93 0.008 100)` |
| Primary | `oklch(0.46 0.11 155)` | `oklch(0.78 0.12 155)` |
| Accent | `oklch(0.72 0.14 78)` | `oklch(0.82 0.13 78)` |
| Border | `oklch(0.88 0.012 100)` | `oklch(1 0 0 / 10%)` |
| Card | `oklch(0.995 0.003 95)` | `oklch(0.19 0.012 110)` |

备选方向：

- `Ink / Vermilion / Sage`：石墨黑、朱砂红、鼠尾草绿，更有中文个人品牌感，但红色需要克制使用。
- `Carbon / Lime / Silver`：碳黑、酸橙绿、银灰，更偏工程和开源工具气质，视觉更锋利。

当前建议先采用 `Editorial Tech / Warm Graphite`，因为它更适合“AI 技术顾问 + 开源作者 + 内容分享者”的混合身份，既专业，也不会过于冷硬。

## 首页调整范围

第一阶段只调整首页相关文件，当前已完成并确认效果：

- `src/app/home/page.tsx`
- `src/app/home/components/hero-section.tsx`
- `src/app/home/components/stats-section.tsx`
- `src/app/home/components/projects-section.tsx`
- `src/app/home/components/skills-section.tsx`
- `src/app/home/components/timeline-section.tsx`
- `src/app/home/components/services-section.tsx`
- 必要时少量调整 `src/app/globals.css` 中的主题变量，但要避免影响其他页面过大。

## 全站适配范围

第二阶段将把首页已确认的视觉方案适配到全站，范围包括：

- 公共框架：`src/components/navbar.tsx`、`src/components/footer.tsx`、`src/components/sections/contact-section.tsx`、共享联系卡片与二维码弹层。
- 内容页面：`src/app/about/page.tsx`、`src/app/projects/page.tsx`、`src/app/blog/page.tsx`、`src/app/blog/[slug]/page.tsx`。
- 服务页面：`src/app/services/page.tsx`。
- 管理页面：`src/app/students-management/page.tsx` 及其客户端组件。管理页面应更偏工作台风格，使用同一色彩 token，但保持信息密度和表单可用性。
- 全局主题变量：必要时调整 `src/app/globals.css`，把当前首页中反复使用的暖白、石墨、翡翠绿、琥珀金沉淀为全局主题变量，减少页面内重复任意值。

## 技术方案

1. 先抽象视觉 token
   - 优先使用 Tailwind 任意值或局部 className 调整首页。
   - 首页试点确认后，可以把稳定色板沉淀到 `globals.css` 的主题变量中，让公共组件和其他页面自然继承。

2. 先调整 Hero 区
   - 替换头像蓝色光晕为翡翠绿与琥珀金的细腻环形层次。
   - 姓名颜色从蓝色改为石墨/翡翠强调。
   - 打字条从蓝色发光胶囊改为更克制的编辑化信息条。
   - 主按钮改为翡翠绿实心按钮，次按钮保持低对比 outline。
   - 背景从蓝色径向渐变改为细微纸感、网格或柔和光带，避免装饰性大色块。

3. 再调整首页中段
   - Stats 数字与图标使用翡翠绿/琥珀金建立节奏。
   - 项目卡片减少蓝色 hover，改用边框、阴影、顶部细线或角标形成可感知反馈。
   - 技能标签区分“分类标签”和“技能项”，避免所有标签长得一样。

4. 最后调整时间线、服务和联系入口
   - 时间线颜色从蓝绿橙混合改为翡翠绿、琥珀金、石墨灰的连续系统。
   - 服务卡片和联系入口跟随首页视觉语言，但不改文案。

5. 全站推广
   - 第一批：公共框架（导航、页脚、联系区）和全局主题变量。
   - 第二批：关于、项目、博客等内容页。
   - 第三批：服务页。该页面内容较多，需要分段适配卡片、CTA、优惠券、二维码和流程说明。
   - 第四批：学生管理页。保持工具型页面的信息密度，不做营销化布局。

## 拆分与验证流程

1. 创建并确认需求文档。
2. 只实现首页 Hero 区视觉试点。
3. 停下来总结代码和视觉变化，等待 review。
4. 确认后继续调整首页其他 section。
5. 首页全部调整完成后运行：
   - `npm run lint`
   - `npm test`
   - `npm run build`
6. 首页效果确认后，补充全站扩展需求，逐批调整其他页面。
7. 每完成一个批次停下来总结代码与视觉变化，等待 review 后再进入下一批。

## 验收标准

- 全站不再以蓝色渐变作为主视觉。
- 全站文案完全保持不变。
- 页面 section 之间应保持编辑化的连续阅读节奏，避免出现过于明显的横向色块分界。
- 浅色和深色模式都能正常阅读，按钮、链接、焦点态有明确可访问的对比度。
- 移动端和桌面端布局不出现文字重叠、按钮挤压、卡片内容溢出。
- 每个适配批次完成后 lint、test、build 均通过。

## 已确认事项

- 确认采用 `Editorial Tech / Warm Graphite` 作为首页试点方向。
- 允许为首页引入轻量背景纹理或 CSS-only 纸感噪点。
- 允许在不改文案的前提下微调首页 section 间距和卡片布局。
- 保留深色模式，浅色模式与深色模式都需要同步打磨。
- 首页试点效果已确认，当前方案适配到整个网站。
- 头像周围保留纯绿色光圈，外层绿色光晕保持偏淡，不再使用多色渐变。
- 首页 section 分隔需要弱化：不使用明显横向色块切分，优先通过轻边界、留白、标题节奏和卡片层级区分内容。
